# Mac client — ScreenerBar

The macOS side of Mail Screener. Talks to the `screenerd` REST API over the LAN
(no Apple private APIs). The daemon also serves the same UI in a browser at its
API root — this app is the native menu-bar equivalent.

- **ScreenerKit** — shared Swift package: typed async REST client, DTOs, Keychain
  helper.
- **ScreenerBar** — `MenuBarExtra` app: live status, list view/edit, wildcard
  suggestions, settings (server URL + token).

## Build

The `.xcodeproj` is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)
from `project.yml` and is **not** committed.

```bash
cd mac
xcodegen generate      # -> ScreenerBar.xcodeproj
open ScreenerBar.xcodeproj
# build & run in Xcode (⌘R). It appears as a mail (envelope) icon in the menu bar.
```

ScreenerKit on its own:

```bash
cd mac/ScreenerKit
swift test
```

## First run

Open the menu-bar window → **Settings**:

- **Base URL** — e.g. `http://iphnnas:18443` (host + port only, no path).
- **API Token** — the daemon's `API_TOKEN` (stored in the Keychain, not on disk).

Then **Test Connection**. The token is the same value as in the daemon's `.env`.

## Notes

- Min macOS 14. Menu-bar-only (`LSUIElement`), no Dock icon.
- App Sandbox is **off** (personal app). LAN plain-HTTP is allowed via
  `NSAllowsLocalNetworking`.
